   **switchTo('openrouter')** flow (per D-01, D-03, D-09, D-10):
   a. Get the OpenRouter API key from keychain via `getOpenRouterKey()`.
   b. If no key found, prompt the user for it using `@clack/prompts` `text()` with message `'Enter your OpenRouter API key:'` and `placeholder: 'sk-or-...'`. Validate it is non-empty. Save it via `saveOpenRouterKey(key)`.
   c. Check if `ANTHROPIC_API_KEY` is set in current environment (`process.env.ANTHROPIC_API_KEY`). If set and non-empty, save it to keychain via `saveAnthropicKey()` so it can be restored later (per D-10). Log: `'Saved your Anthropic API key to keychain for later restoration.'`
   d. Get profile path via `getProfilePath()`. Read profile via `readProfile(profilePath)`.
   e. Build the OpenRouter env var block via `buildOpenRouterBlock(openRouterKey)`.
   f. Inject the block via `injectBlock(profileContent, block)`. Write via `writeProfile(profilePath, newContent)`.
   g. Set mode via `setMode('openrouter')`.
   h. Print success with chalk green: `'Switched to OpenRouter. Open a new terminal for changes to take effect.'`
   i. On Windows (`process.platform === 'win32'`), additionally warn about execution policy: `'Note: Ensure PowerShell execution policy allows running profile scripts: Set-ExecutionPolicy RemoteSigned -Scope CurrentUser'` (per Pitfall 5).

   **switchTo('anthropic')** flow (per D-02, D-03):
   a. Get profile path and read profile.
   b. Check if the user has a stored Anthropic API key via `getAnthropicKey()`.
   c. If they have an original key: build a restore block via `buildAnthropicRestoreBlock(anthropicKey)` and inject it. If the restore block is null (no key), just remove the block via `removeBlock(profileContent)`.
   d. Write profile, set mode to `'anthropic'`.
   e. Print success with chalk green: `'Switched to Anthropic (native). Open a new terminal for changes to take effect.'`

   Import chalk for colored output. Import `* as p from '@clack/prompts'` for the API key prompt.

      - `'Model mappings:'` header, then for each tier (opus, sonnet, haiku): `'  {tier}: {model || "(not set)"}' `

   a. Import `Command` from `commander`.
   b. Create program with `name('claude-switcher')`, `description('Switch Claude Code between Anthropic and OpenRouter APIs')`, `version('1.0.0')`.
   c. Add option `--use <provider>` with description `'Switch to provider (anthropic or openrouter)'`. Validate choices are only `anthropic` or `openrouter`.
   d. Add option `--status` with description `'Show current mode and configuration'`.
   e. Parse `process.argv`.
   f. Routing logic:
      - If `opts.use` is set: call `switchTo(opts.use)` from `./commands/switch.js`.
      - Else if `opts.status` is set: call `showStatus()` from `./commands/status.js`.
      - Else (no flags, per D-11): call `runMenu()` from `./commands/menu.js`.
   g. Wrap the routing in try/catch. On error, print error message in chalk red and `process.exit(1)`.

4. Create `bin/claude-switcher.js`:

